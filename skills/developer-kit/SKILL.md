## 使用场景

### 场景 1：新项目启动
```
用户: 我想做一个 React 项目
AI: 
  1. [react-expert] 设计组件架构
  2. [typescript] 配置 tsconfig.json
  3. [testing-patterns] 设置 Jest + Cypress
  4. [git-essentials] 初始化 Git + 分支策略
```

### 场景 2：代码审查
```
用户: 帮我审查这个 PR
AI:
  1. [github-repo-analyzer] 分析仓库结构
  2. [typescript] 检查类型安全
  3. [testing-patterns] 验证测试覆盖
  4. [react-expert] 评估组件设计
```

### 场景 3：重构优化
```
用户: 这段代码需要重构
AI:
  1. [typescript] 提取类型定义
  2. [react-expert] 优化组件结构
  3. [testing-patterns] 补充测试
  4. [git-essentials] 规划重构分支
```

## 套件协同

技能之间自动协作：

- **react-expert + typescript**: React 组件自动生成 TypeScript 类型
- **testing-patterns + react-expert**: 为组件生成测试模板
- **git-essentials + github-repo-analyzer**: Git 提交分析 + PR 建议
- **typescript + testing-patterns**: 类型驱动的测试用例
