# awesome-closet-stylist

一个轻量级的 agent skill，用于维护个人衣橱、回答与衣橱相关的问题，并基于已有衣物用自然语言推荐穿搭。

## 功能概览

这个 skill 可以帮助 agent：
- 维护持久化衣橱
- 新增、更正、更新或删除衣物条目
- 按类别、季节、颜色或风格回答衣橱查询
- 基于用户现有衣物推荐穿搭
- 在上一套推荐的基础上微调，而不是全部重来
- 使用图片作为辅助输入，帮助识别和录入衣物

## 设计目标

这个 skill 的设计刻意保持务实和克制：
- 使用自然语言，而不是暴露原始 JSON 或数据结构
- 将 `user/wardrobe.json` 视为事实来源
- 仅在必要时最多追问一个关键问题
- 优先考虑实穿性，其次是场合，最后才是风格
- 绝不编造衣橱中不存在的衣物
- 在这个 MVP 中，不把衣橱缺口自动转化为购物建议

## 示例用例

- "我刚买了一件深灰色羊毛大衣，帮我记进衣橱。"
- "帮我看看现在衣橱里适合春天穿的外套有哪些。"
- "明天 18 度，有点风，我要去见客户，帮我用我现有的衣服搭一套。"
- "刚才那套别动太多，把鞋换得更轻松一点。"

## 初始化指导

当 Agent 第一次使用这个 skill，或读取到 `user/wardrobe.json` 中 `items` 为空时，应该先把当前状态视为”未初始化衣橱”，邀请用户上传或描述衣物数据，再进入常规查询与推荐流程。

推荐交互方式：
- 先用自然语言说明这个 skill 可以帮用户记录衣物、查询现有衣橱、以及基于现有衣物推荐穿搭
- 明确邀请用户上传衣物照片，或直接描述常穿衣物
- 如需基于实时天气搭配，可引导用户在 `user/config.json` 中填写天气 API key，并说明数据只保存在本地
- 可优先推荐更容易配置的天气服务（如和风天气）；如果用户嫌麻烦，也可以退回由 Agent 通过浏览器搜索天气
- 不要一上来连续追问很多字段，也不要把对话变成表单填写
- 优先邀请用户先提供最常穿、最好搭配、最有代表性的几件衣物
- 如果用户愿意，也可以让用户按类别逐步补充，例如外套、上装、下装、鞋
- 如果用户发来图片，可以把图片当作辅助输入来识别和录入衣物

推荐的开场示例：
- “你的衣橱现在还是空的。我可以先帮你建一个可用的衣橱，你可以直接发几件常穿衣物的照片，或者先告诉我最常穿的外套、上装、下装和鞋，各一两件就行。”
- “我们先做一个轻量初始化吧。你可以上传衣物图片，或者先说几件你最常穿、最有代表性的衣物，我来帮你整理进衣橱。”

初始化阶段的原则：
- 先追求建立最小可用衣橱，而不是一次性收集完整信息
- 用户只提供部分信息时，也应优先录入可识别的最小记录
- 只有在缺少信息会明显影响录入或推荐结果时，才追问一个关键问题
- 完成初始化后，可以主动告诉用户之后可以直接问“我有什么适合春天的外套”或“明天 18 度怎么搭”这类问题

## 数据模型

衣橱数据存储在 [user/wardrobe.json](user/wardrobe.json) 中。

当每件衣物包含以下最小可用信息时，这个 skill 的效果最佳：
- `id`
- `category`
- `name`

可选字段包括：
- `color`
- `seasons`
- `style`
- `material`
- `warmth`
- `notes`
- `image_refs`

## 评估覆盖范围

仓库内的评估用例覆盖：
- 新增衣物
- 查询衣橱条目
- 基于现有衣橱进行穿搭推荐
- 微调上一套穿搭
- 在衣橱为空或接近为空时的追问处理
- 对含糊维护请求的澄清处理
- 图片辅助录入行为
