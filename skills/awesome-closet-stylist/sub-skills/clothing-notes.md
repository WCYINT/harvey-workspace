# clothing-notes

衣物备注异步维护规约。负责识别对话中针对特定衣物的评价或标注信号，并静默更新对应衣物的 `notes` 字段。

---

## 触发条件

以下类型的表达应触发衣物 notes 更新：

| 信号类型 | 示例表达 |
|---|---|
| 穿着体验 | "那件白衬衫穿着有点紧"、"这双鞋走路时间长了会磨脚" |
| 搭配限制 | "那件格子外套很难搭下装"、"这条裙子只适合正式场合" |
| 养护提醒 | "那件毛衣要手洗"、"皮带容易褪色要注意" |
| 主观感受 | "那件大衣我不太喜欢了"、"这双靴子是我最常穿的" |
| 使用频率 | "那件外套我基本不穿了"、"这件基本每周都在穿" |
| 临时标记 | "帮我备注一下那件蓝裤子要修改裤脚"、"记一下这件还没干洗" |

---

## 目标衣物定位

1. 优先从对话当前上下文中定位用户在说哪件衣物（尤其是刚刚提到过的）。
2. 若上下文中存在多件候选，尝试通过类别、颜色等属性唯一确定。
3. 若无法确定目标，才追问一次："你说的是哪件？"

---

## 更新行为

1. **静默写入**：在生成主回复的同时，将信号内容以自然语言形式追加或更新到目标衣物的 `notes` 字段，不打断对话。
2. **追加而非覆盖**：默认将新备注追加到现有 notes 后，不删除历史备注，除非新内容明确推翻旧备注。
3. **格式**：`notes` 字段为 `{ date, text }[]` 数组格式（与 `wardrobe-crud.md` 对齐），每条记录包含 `date`（`YYYY-MM-DD`）和 `text`；只允许追加新条目，不得修改或删除历史条目。
4. **不做确认**：不需要每次写入后告诉用户"已记录"，保持对话流畅；除非用户明确要求确认，或备注内容较为重要（如"要扔掉"这类不可逆判断）。

---

## 示例

**场景一**：用户提到穿着体验
> 用户："那件格子外套真的很难搭，基本上搭什么都怪"
> → 定位到格子外套，在 `notes` 追加："难以搭配，不建议作为推荐优先选项"
> → 主回复正常进行，不提写入动作

**场景二**：用户明确要求记录
> 用户："帮我备注一下那件卡其风衣要去干洗"
> → 定位到卡其风衣，写入 `notes`："需要干洗"
> → 主回复可简短确认："好，已经记下来了。"（此类明确指令可以确认）

**场景三**：目标衣物不明确
> 用户："这件有点紧"
> → 无法从上下文定位，追问："你说的是哪件？"

## Hooks 自动整理

当本次 session 首次识别到衣物反馈时，按 `SKILL.md` 中"Hooks 自动整理"章节的判定流程处理。

判定与提示在 session 级别只执行一次，由首个触发的 sub-skill 负责；若另一个 sub-skill 后续也触发，不再重复提示。
