### 20:38 OpenClaw Context 管理最佳实践（重大发现）

**问题根源澄清：**
- Session context 达到 204800/204800 是消息历史累积，不是 memory-lancedb-pro 问题
- `compaction.mode = "safeguard"` **本身就是 auto-compaction**（不是"只保护不压缩"）
- 误解来源：之前认为 safeguard = 不作为，实际 OpenClaw 文档确认 safeguard = 接近 window 时自动触发 summarization

**memory-lancedb-pro 机制确认：**
| 功能 | 默认状态 | 对 context 影响 |
|------|---------|---------------|
| autoRecall | OFF | 不注入记忆到 context |
| autoCapture | ON | 存储记忆到 LanceDB，不占 context |
| Session Summaries | OFF | 不影响 |

**OpenClaw 内置 context 管理机制：**
1. **Auto-compaction (safeguard)**：接近 context window 时自动压缩旧对话为摘要
2. **Session pruning**：可配置 `contextPruning.mode = "cache-ttl"` 修剪旧 tool results
3. **Manual compaction**：`/compact` 命令手动触发
4. **Session reset**：`/new` 或 `/reset` 完全清空

**最佳实践（官方推荐）：**
1. 日常：感觉对话变慢 → `/compact` 主动压缩
2. 切换话题：使用 `/new` 全新开始
3. 超长会话（>50轮）：`/compact` 后继续
4. 可选配置：启用 `contextPruning` 减少 tool result 累积

**配置现状（无需修改）：**
- `compaction.mode = "safeguard"` ✅ 已是 auto-compaction
- `contextPruning` 未配置（可选，可启用但当前够用）
- `plugins.slots.contextEngine` 未设置（使用 legacy 默认引擎）

### 论文格式标准
**西安交通大学MBA论文格式要求（2026-03-28收到）**

来源文件: 西安交大MBA论文格式要求(1).pdf

**论文结构（11个部分）**：封面 → 题名页 → 摘要 → 目录 → 符号表 → 正文 → 致谢 → 参考文献 → 附录 → 研究成果 → 声明

**摘要要求**：
- 硕士: ~600汉字，突出新见解
- 博士: ~1000汉字，突出创新点
- 关键词: 3-5个

**正文要求**：
- 硕士: 40-60页
- 必须有绪论和结论

**参考文献要求**：
- 硕士: ≥30篇（含期刊≥20、国外≥10）
- 时间: 近5年为主

**论文类型**: 工程/项目管理（j类）- 适用于插电混动汽车密封条质量改进
