# Thesis Design Review - 论文设计审查

## 审查维度

| 维度 | 评分 | 说明 |
|------|------|------|
| **研究框架** | 0-10 | 逻辑结构、章节安排 |
| **方法论选择** | 0-10 | 工具适切性、数据支撑 |
| **论证结构** | 0-10 | 假设→论证→结论 |
| **创新性** | 0-10 | 理论/实践贡献 |
| **可读性** | 0-10 | 表达清晰度 |
| **完整性** | 0-10 | 边界条件、局限性 |

## 审查流程

### Step 1: 研究框架评估

问题：
- 章节安排是否遵循"现状→问题→原因→方案→效果"逻辑？
- 各章节比例是否合理？
- 是否有逻辑跳跃？

### Step 2: 方法论评估

问题：
- 帕累托图、鱼骨图、DOE 是否适合本研究？
- 是否有更优的工具选择？
- 方法论链条是否完整？

### Step 3: 论证结构评估

问题：
- 假设是否清晰？
- 论证是否充分？
- 结论是否呼应假设？

### Step 4: 改进建议

根据评分，给出：
- 最高分维度：保持
- 最低分维度：重点改进
- 改进路径

## 使用方法

告诉我想审查的论文设计维度，我会：
1. 评估各维度得分 (0-10)
2. 解释得分原因
3. 给出改进建议
